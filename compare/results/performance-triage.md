goal. It records both successful optimizations and rejected changes so the

## Next Candidate

Continue performance triage on the remaining focused watch items only when a
new concrete hypothesis is available:

- LogNormal: wrapper dispatch, normal generation, Rust algorithm shape,
  `std.math.exp`, optimized float mode, `exp2`, f32 vector width,
  general-purpose `expm1 + 1`, and branchy hybrid `expm1` have all been ruled
  out as defaults; future work needs a sound exact-`exp` transform/codegen
  improvement or an explicitly parameter-limited opt-in. See
  `lognormal-transform-notes.md`.
- OpenClosed f64: buffer sizes, int-add expression shape, bitcast, in-place,
  vector, and invalid raw-byte variants are exhausted; future work needs a new
  exact `(0, 1]` endpoint-grid construction. See
  `openclosed-endpoint-notes.md`.
- SIMD distributions: f32x8 repair is useful evidence, but both standard and
  parameterized vectorbench slice-fill rows failed to beat current scalar
  lane-fill defaults; future work needs a genuinely dense SIMD candidate that
  meets `simd-distribution-kernel-notes.md`.
